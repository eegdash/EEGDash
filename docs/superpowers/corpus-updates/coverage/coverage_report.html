Where ntimes comes from now — and what it cost.

Before, every resolved n_times came from one source (the sidecar) and BrainVision needed a full read_raw_brainvision. Now the work fans out — and the new binary_parser band is pure git-annex-key arithmetic.

sidecar_arithmetic · 137,857 · 79.9%
binary · 15,347 · 8.9%
unresolved · 19,228 · 11.1%
sidecar arithmetic — round(sfreq × RecordingDuration) binary_parser — .eeg size from the annex key, 0 bytes fetched no cheap path on a shallow clone

The 8.9% blue band is the whole point. All 15,347 of those are BrainVision .vhdr recordings whose sample count was computed as datafile_bytes ÷ (nchans × dtype_bytes) — read straight from the git-annex MD5E-s<size> key. The old pipeline would have fetched each .eeg from S3 and run MNE; the new one fetches nothing and gets the byte-exact answer (certified equal to MNE). Same recovery, zero signal I/O.

Why MEF3 fell back to 4%. The header reader I added does extract number_of_samples — but only when the .tmet is present. On a shallow clone it's a git-annex pointer, so MEF3 (and SNIRF, both binary-header formats) drop to sidecar arithmetic, which most of these datasets lack. VHDR is the exception precisely because its count needs only the companion's size, not its contents. Closing this means either a full clone, a ranged header fetch, or enumerating the still-uncovered exotics (.gz, .apr, .apx).

The remote tier — the gap closed in a few bytes.

With EEGDASH_REMOTE_HEADERS=1, the formats whose binary is an annex pointer get a single ranged read of just the header — never the signal. Re-digested on real OpenNeuro gap datasets, every one went to 100%.

EDF/BDF: 256 bytes. The annotation-safe record-count formula needs only the 256-byte main header. MEF3: one ~16 KB .tmet (the metadata file; the compressed .tdat is never touched). SNIRF: ~196 KB of HDF5 metadata — h5py over a Range-backed file reads only the object headers (0.24% of an 80 MB file), no signal. Every value is byte-exact and provenance-stamped remote_header. Projected corpus-wide, this lifts ntimes 88.9% → high-90s while fetching, in total, a rounding error of the data.

How this was made

Enumerated via OpenNeuro GraphQL (eeg/ieeg/meg/nirs) + NEMAR catalogue → 696 datasets. Each shallow-cloned (git clone --depth 1, GIT_LFS_SKIP_SMUDGE=1) and digested with the cheap-exact resolver, then aggregated by coverage_report.py. 685 digested (11 clone/digest stragglers). No raw signal read at any point.

The win, precisely

  • duration_seconds: 0% → 89.7%
  • 15,347 VHDR n_times from the annex key — 0 fetches
  • byte-exact, MNE-certified; provenance-stamped
  • VHDR resolution path ~14× faster

What's still open

Shallow-clone binary-header formats (MEF3, SNIRF) need the file present — a full clone, a ranged header fetch, or sidecar duration. Exotic .gz / .apr / .apx have no parser yet. All are tracked in the Phase-3 plan.
