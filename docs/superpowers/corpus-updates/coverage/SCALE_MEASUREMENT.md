# Scale measurement: what technical metadata is missing across all datasets

Generated by `scripts/ingestions/coverage_report.py` over the local
`scripts/ingestions/digestion_output/` corpus — **289 datasets, 39,022 records**
spanning EEG / iEEG / MEG / fNIRS. This is the **BEFORE** state (records digested
with the pre-resolver pipeline). Raw numbers: `coverage_before.json` /
`coverage_before.txt`.

## Headline coverage (resolved / total)

| Field | Coverage | Missing |
|-------|----------|---------|
| sampling_frequency | 97.54 % | 960 |
| nchans | 97.39 % | 1 020 |
| **ntimes** | **83.78 %** | **6 330** |
| ch_names | 94.23 % | 2 253 |
| duration_seconds | 0.00 % | 39 022 (field did not exist) |

Before this work, **every** resolved `ntimes` came from a single source
(`mne_bids` sidecar arithmetic); when that path could not run, `ntimes` was left
`None`. There was no `duration_seconds` field at all.

By modality, `ntimes` coverage: eeg 84.5 %, fnirs 94.6 %, ieeg 80.6 %, meg 79.6 %.

## How the cheap resolver closes these gaps

- **Universal (all formats), Phase 1:** `ntimes = round(SamplingFrequency ×
  RecordingDuration)` is now promoted into the sidecar cascade step, so any record
  whose modality JSON carries `RecordingDuration` gets `ntimes` + `duration_seconds`
  with **zero** binary access — independent of format. This lifts the long tail
  across `.mefd`, `.ds`, `.fif`, `.con`, etc.
- **`.vhdr` (7 337 records, the largest single gap), Phase 2:** `ntimes` now comes
  from `[Common Infos] DataPoints` or `.eeg` file-size arithmetic (the `.eeg` size
  is read from the git-annex key — zero signal bytes), removing the
  `read_raw_brainvision` fallback entirely. Certified equal to MNE by
  `tests/validation/test_mne_equivalence.py`.
- **`.snirf` / `.set`, Phase 2:** header-only `n_times` (`dataTimeSeries.shape` /
  `EEG.pnts` via `variable_names=['EEG']`), no full-array reads.
- `duration_seconds` is now persisted (derived `ntimes / sfreq` or sidecar
  `RecordingDuration`), so its 0 % becomes ~the `ntimes` coverage or better.

## Reproducing at NEMAR + OpenNeuro scale

1. Enumerate via the existing source APIs (OpenNeuro GraphQL `latestSnapshot.summary`;
   NEMAR `data.nemar.org/?format=json`) — zero file reads.
2. `2_clone.py` each dataset: `git clone --depth 1` + `GIT_LFS_SKIP_SMUDGE=1`
   (sidecars + annex pointers only — no signal).
3. `3_digest.py` to produce `*_records.json` (cheap resolver runs here; the
   file-size tier uses the annex key, so VHDR/EDF `ntimes` needs no fetch).
4. `python coverage_report.py <output_dir> --json after.json` and diff against
   `coverage_before.json` to quantify the lift.
